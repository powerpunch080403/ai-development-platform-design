# Identity and Approval

## 분리할 신원

- Human Identity: 사용자 계정과 승인 권한
- Node Identity: 개인 서버의 장치 신원
- Worker Identity: Node 내부 Worker 실행 신원

토큰과 권한을 서로 공유하지 않는다.

## 대화로 처리

- 제품 방향
- 기술 선택
- 우선순위와 기능 범위
- 설계 대안

Owner가 Decision Proposal로 정리한다.

## 버튼 승인

- Node 연결
- 권한 상승
- 비밀 접근
- 대량 삭제
- 기본 브랜치 병합
- 외부 배포
- 유료 API 사용
- 보호 영역 수정
