# Agent Instructions

이 저장소는 제품 및 시스템 설계 문서 저장소다.

1. 사용자가 명시하지 않은 제품 결정을 확정하지 않는다.
2. 확정되지 않은 내용은 `status: proposed` 또는 Open Questions에 기록한다.
3. 중요한 아키텍처 변경은 ADR을 작성한다.
4. 과거 ADR을 조용히 수정해 결정을 지우지 않는다.
5. 관련 링크와 Open Questions를 함께 갱신한다.
6. Discord를 제품 인터페이스로 다시 제안하지 않는다.
7. 사용자가 Worker를 직접 조작하는 UX를 제안하지 않는다.
8. Human, Node, Worker Identity를 혼합하지 않는다.
9. 개인 Node를 중앙 Authority의 동등한 DB Writer로 만들지 않는다.
10. 수정 후 변경 파일, 새 결정, 미해결 질문을 요약한다.
